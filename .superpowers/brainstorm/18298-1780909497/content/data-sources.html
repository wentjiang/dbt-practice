What are the multiple data sources?

This determines how the ingestion (Bronze) layer is structured

A

CSV / flat files

Static files simulating exports from different systems — e.g. one CSV for transactions, another for property attributes from a different vendor

B

Multiple PostgreSQL schemas / tables

Simulate different source systems as separate schemas in the same Postgres instance — e.g. src_agent_system and src_listings_portal

C

REST API (simulated)

A small mock API (e.g. FastAPI in Docker) that returns property data — ingested via a Python script into the raw layer

D

Mix: CSV + PostgreSQL schemas

Most realistic for learning — one source as CSV files (e.g. property attributes from a data provider), another as a live DB schema (e.g. transaction system)
